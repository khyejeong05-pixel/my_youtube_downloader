# 제품 요구사항 정의서 (PRD) - 한국어 유튜브 다운로더 (YouTube Downloader)

본 문서는 GitHub의 [Hitomi-Downloader](https://github.com/KurtBestor/Hitomi-Downloader) 프로젝트를 분석하고, 이를 벤치마킹하여 개발할 **한국어 버전 유튜브 다운로더 프로그램**의 제품 요구사항을 정의합니다.

---

## 1. 제품 개요 (Product Overview)
본 제품은 사용자가 유튜브 동영상 및 오디오를 손쉽고 빠르게 다운로드할 수 있는 데스크톱 GUI 애플리케이션입니다. Hitomi-Downloader의 강력하고 직관적인 다운로드 관리 메커니즘을 벤치마킹하여, 한국어 사용자 환경에 맞춘 직관적인 UI/UX와 고성능 다운로드 기능을 제공합니다.

### 1.1 핵심 가치
* **사용 용이성**: 복잡한 설정 없이 유튜브 URL 입력만으로 즉시 다운로드 가능.
* **고성능 및 신뢰성**: 멀티스레드 기반 속도 향상 및 안정적인 다운로드 (yt-dlp 엔진 사용).
* **시각적 완성도**: 모던한 다크 모드 지원 및 직관적인 진행 상황 피드백.
* **포터블 배포**: 설치가 필요 없는 단일 실행 파일(.exe) 형태의 포터블 배포 지향.

---

## 2. 사용자 분석 및 시나리오 (Target Audience & Use Cases)

### 2.1 대상 사용자
* 유튜브의 고화질 동영상(1080p, 4K 등)을 소장하고 싶은 사용자.
* 유튜브 동영상에서 오디오(MP3)만 추출하여 오프라인으로 듣고 싶은 사용자.
* CLI(명령줄) 도구인 yt-dlp의 사용법이 어렵게 느껴져 편리한 GUI를 원하는 사용자.

### 2.2 핵심 사용 시나리오
1. **단일 영상 다운로드**: 사용자가 유튜브 링크를 복사하여 프로그램에 붙여넣고, 원하는 화질(예: 1080p) 또는 오디오 포맷(MP3)을 선택한 뒤 다운로드 버튼을 누릅니다.
2. **다중 작업 큐 관리**: 여러 개의 동영상 링크를 동시에 추가하여 순차적으로 다운로드하고, 개별 작업의 진행률, 다운로드 속도, 남은 시간(ETA)을 실시간으로 확인합니다.
3. **클립보드 감시**: 프로그램이 켜져 있는 동안 유튜브 링크가 클립보드에 복사되면 자동으로 감지하여 입력란에 채워줍니다.

---

## 3. 핵심 기능 요구사항 (Key Features)

### 3.1 다운로드 엔진 (Core Downloader Engine)
* **yt-dlp 통합**: 최신 유튜브 다운로드 알고리즘 대응 및 유지보수를 위해 내부 엔진으로 `yt-dlp` 사용.
* **포맷 및 화질 선택**:
  * 비디오: 최고 화질(Best Quality), 1080p, 720p, 480p 등 해상도별 다운로드 지원 (MP4 포맷).
  * 오디오: 고음질 오디오 추출 및 MP3/M4A 포맷 변환 지원.
* **진행 상태 정보**: 다운로드 속도(MB/s), 전체 크기 대비 받은 크기, 진행률(%), 남은 시간(ETA) 실시간 표시.
* **작업 제어**: 다운로드 중 취소(Cancel) 및 일시 정지/재개(Pause/Resume) 기능 제공.

### 3.2 작업 큐 및 목록 관리 (Task Queue & List Management)
* **다중 다운로드 지원**: 여러 개의 다운로드 작업을 목록에 추가하고 큐(Queue) 형태로 관리.
* **개별 작업 상태 표시**: 각 작업의 썸네일, 제목, 해상도, 진행 표시줄(ProgressBar) 및 상태(대기 중, 다운로드 중, 완료, 오류)를 직관적으로 표시.
* **목록 정리**: 완료된 작업 제거, 전체 완료 목록 지우기 기능.

### 3.3 사용자 편의 기능 (Convenience Features)
* **클립보드 자동 모니터링**: 클립보드에 유튜브 URL(`youtube.com` 또는 `youtu.be`)이 감지되면 자동으로 입력창에 주소를 추가하는 옵션 제공.
* **다운로드 경로 설정**: 파일이 저장될 기본 디렉토리 선택 기능 (기본값: 사용자 폴더의 다운로드 또는 동영상 폴더).
* **완료 알림**: 다운로드 작업 완료 시 시각적인 알림 또는 소리 알림.

## 4. UI/UX 디자인 요구사항 (User Interface)

Hitomi-Downloader의 레이아웃을 계승하면서도, 현대적인 디자인 트렌드를 반영한 **Premium Dark Theme**를 적용합니다.

* **메인 레이아웃**:
  * **상단**: URL 입력창, 다운로드 옵션 선택(화질/포맷), 다운로드 시작 버튼.
  * **중앙**: 현재 진행 중인 작업 및 대기 작업 목록 (리스트 뷰).
  * **하단**: 상태 표시줄 (현재 전체 다운로드 속도, 저장 폴더 열기 버튼, 설정 버튼).
* **스타일링**:
  * **테마**: 기본 다크 테마(Sleek Dark Theme) 적용, 가독성이 뛰어난 폰트 사용.
  * **색상 조합**: 깊은 차콜/네이비 배경(#1E1E2E), 강조색으로 일렉트릭 블루 또는 퍼플(#7F5AF0)을 사용하여 프리미엄 감성 제공.
  * **애니메이션**: 버튼 호버 시 미세한 밝기 변화 및 부드러운 진행바 애니메이션 적용.
* **완벽한 한국어 지원**: 모든 레이블, 버튼, 경고 메시지, 오류 정보 등을 자연스러운 한국어로 제공.

---

## 5. 기술 스택 (Technical Stack)

* **언어**: Python 3.10+
* **GUI 라이브러리**: `customtkinter` (Tkinter의 모던 래퍼, 미려한 다크 모드 기본 지원) 또는 `PyQt5`/`PySide6`. 
  * *결정*: 가볍고 빠르며 미려한 UI를 쉽게 구현할 수 있는 **CustomTkinter** 채택.
* **다운로드 핵심**: `yt-dlp` (Python 라이브러리 형태로 내장 또는 subprocess 호출).
* **후처리**: 오디오 변환 및 포맷 병합을 위해 `ffmpeg` 활용 (환경 경로에 설치되어 있거나 로컬 바이너리 참조).

---

## 6. 비기능 요구사항 (Non-Functional Requirements)

* **성능**: 대용량 파일 다운로드 시 GUI가 멈추지 않도록 다운로드 작업을 별도의 백그라운드 스레드(Thread)에서 실행.
* **오류 처리**: 네트워크 단절, 잘못된 URL, 비공개 영상 등에 대한 예외 처리 및 한국어 에러 메시지 표시.
* **가벼움**: 의존성을 최소화하여 프로그램 실행 시 딜레이가 없도록 설계.
